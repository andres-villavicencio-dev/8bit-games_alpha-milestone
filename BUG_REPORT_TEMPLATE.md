# Bug Report Template

## Summary
A brief, concise description of the bug (one-line summary).

## Description
A detailed description of the bug, including what happens and what is expected to happen.

## Steps to Reproduce
1. Step-by-step instructions to reproduce the bug
2. Be as detailed as possible
3. Include any prerequisites or setup needed

Example:
1. Navigate to [page/module]
2. Click on [element/button/link]
3. Enter [data/input] in [field/form]
4. Observe [unexpected behavior]

## Expected Result
Describe what should happen when following the steps above.

## Actual Result
Describe what actually happens when following the steps above.

- **Device:** [e.g., Desktop, iPhone X, Samsung Galaxy S10]
- **OS:** [e.g., Windows 10, macOS Big Sur, Android 11, iOS 14]
- **Browser:** [e.g., Chrome 92, Safari 14, Firefox 90]
- **Version:** [e.g., v1.2.3]
- **Other relevant environment details:**

## Supporting Information
Attach or link to any supporting information that might help diagnose the bug:
- Screenshots or screen recordings
- Error messages or logs
- Network traces (if applicable)
- Sample files or data that trigger the bug

## Severity
How critical is this bug?

- **Critical:** Blocks major functionality, causes data loss, or makes the system unusable
- **High:** Major functionality is impaired but workarounds exist
- **Medium:** Minor functionality issues, affects usability but doesn't block work
- **Low:** Cosmetic issues, spelling errors, minor inconveniences

## Priority
How soon should this be fixed?

- **Urgent:** Must be fixed immediately
- **High:** Should be fixed in the next release
- **Medium:** Can wait until a future release
- **Low:** Nice to have, but not necessary for release

## Possible Solution
If you have an idea of how to fix the bug, describe it here. This is optional.

## Additional Context
Any other information that might be relevant to reproducing or fixing the bug.

## Linked Issues

- Related to #[issue number]
- Part of epic #[epic number]

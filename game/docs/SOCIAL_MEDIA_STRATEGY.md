## 2. Brand Assets Creation

### Digital Assets Required

**Profile Pictures:**
- Primary: Use existing `icon.png` (res://assets/sprites/icon.png)
- Platform variations:
  - Twitter: 400x400 minimum
  - Discord: 128x128 minimum
  - YouTube: 800x800 recommended

**Header Images:**
- Twitter/X: 1500x500 pixels - Game world panorama
- YouTube: 2560x1440 pixels - Full brand layout with title
- Discord: No custom header (server banner: 960x540 optional)

**Game Screenshots Needed:**
1. Main menu scene
2. Overworld exploration
3. Battle encounter
4. NPC dialogue scene
5. Character/party screen
6. Inventory/menu system
7. Day/night cycle (if applicable)
8. Boss battle

**GIF Assets:**
1. Walking animation loop ( protagonist)
2. Combat attack animation
3. Menu opening animation
4. Transition effects (battle start, scene change)
5. Character sprite idle animation

**Video Content:**
1. Announcement trailer (60-90 seconds)
2. Launch trailer (30-60 seconds)
3. Dev diary B-roll (ongoing)

### Asset Creation Guidelines

**Screenshot Best Practices:**
- Capture at native 320x180, upscale for sharing
- Show action moments, not static screens
- Include UI for context in some shots
- Vary environments (overworld, battle, menu)
- Capture during interesting moments (combat, dialogue)

**GIF Recording Tips:**
- Use Godot's built-in recording or OBS
- 5-10 second loops work best
- File size under 5MB for Twitter
- Add subtle text overlays for context
- Maintain 320x180 aspect ratio (upscale 3x)

## 3. Content Calendar Template

### Weekly Content Framework

**Monday - Development Update**
```
Topic: Weekly progress showcase
Content: Screenshot/GIF of a feature in development
Platform: Twitter, Instagram, Discord
Time: 10 AM Pacific
Engagement: Reply to comments, quote-tweet community reactions
```

**Tuesday - Throwback/Lore**
```
Topic: Game lore or classic RPG inspiration
Content: Pixel art appreciation, design philosophy
Platform: Twitter, Reddit
Time: 1 PM Pacific
Engagement: Ask community about their favorite classic RPGs
```

**Wednesday - Work-in-Progress**
```
Topic: Behind-the-scenes development
Content: Raw development screenshot, bug, or challenge
Platform: Twitter, Discord
Time: 11 AM Pacific
Engagement: Ask community for input on design decisions
```

**Thursday - Throwback/Technical**
```
Topic: Technical deep dive or tutorial
Content: Godot tips, pixel art tutorial, game design insights
Platform: YouTube (long-form), Twitter (thread)
Time: 10 AM Pacific
Engagement: Q&A about development process
```

**Friday - Community Spotlight**
```
Topic: Fan content, feedback implementation
Content: Feature fan art, answer questions, thank community
Platform: All platforms
Time: 12 PM Pacific
Engagement: Retweet, feature in Discord, newsletter
```

**Saturday - Weekend/Playtest**
```
Topic: Playtest event or casual streaming
Content: Developer playthrough, community gaming
Platform: Discord, Twitch (if available)
Time: Variable
Engagement: Direct community interaction
```

**Sunday - Weekly Recap**
```
Topic: Summary of week's progress
Content: Thread summarizing all updates from the week
Platform: Twitter, Discord
Time: 5 PM Pacific
Engagement: Thank community, preview next week
```

## 5. Engagement & Community Management

### Daily Engagement Checklist

**Morning (9-10 AM Pacific):**
- [ ] Check Discord overnight messages, reply/engage
- [ ] Check Twitter mentions and replies
- [ ] Review Reddit for game mentions
- [ ] Respond to all comments on previous day's posts
- [ ] Schedule day's content (morning tweet)

**Midday (12-2 PM Pacific):**
- [ ] Post Twitter content
- [ ] Cross-post to Instagram (adapt content)
- [ ] Engage in Discord for 15-30 minutes
- [ ] Check itch.io comments and devlog stats
- [ ] Reply to YouTube comments

**Evening (4-6 PM Pacific):**
- [ ] Post end-of-day content
- [ ] Final Discord presence
- [ ] Schedule any night/weekend content
- [ ] Review day's analytics
- [ ] Update engagement tracking

### Community Response Templates

**Thank You for Feedback:**
```
Thanks for the feedback, [Name]! 🎮

We've noted [specific point] and will consider it as we continue 
development. Community input is crucial to making this the best 
experience possible.

Join our Discord to follow the progress: [link]
```

**Answering "When is it releasing?":**
```
We're targeting [quarter/year] - want to make sure everything 
is polished before we release!

Wishlist/follow to get notified when we announce the exact date.
```

**Addressing Bugs/Issues in Demos:**
```
Sorry you encountered that! 🐛 Could you share:
- Your OS (Windows/Mac/Linux)
- Where in the game it happened
- Steps to reproduce (if you can)

We'll investigate right away. Thanks for helping improve the game!
```

**Welcoming New Community Members:**
```
Welcome to the community, [Name]! 🎉

We're building an 8-bit RPG inspired by the classics. Feel free
to explore the channels, share your thoughts, and join the 
conversation!

What's your favorite classic RPG?
```
